# 17 — 桌面小组件：不写一行安卓代码，把两行字钉在主屏上

> 后续我们做了原生安卓小组件（写 Kotlin、嵌在自己的 APK 里），见 [22-native-widgets.md](22-native-widgets.md)。本篇的 Tasker 方案仍然有效，适合不想碰安卓开发的情况。

## 这是什么

手机桌面上一块小卡片，每天显示「在一起第 X 天 · 今日签等级」和「宜：…… · 幸运物：……」。全程不碰安卓开发：**服务端一个只读接口 + Tasker Widget v2**，一个上午从零到上墙（含踩坑时间）。

适合放小组件的内容有个筛选标准：**每天会变、但不会变成焦虑源**。我们排除了实时情绪值（盯着看会变成压力），选了每日签——它有一个额外的好处，见下一节。

## 设计：同源同签 + 信封原则

前端本来就有「抽签盒」功能：纯前端 `fortune.js`，种子 = `日期|人名|盐`，seeded PRNG（mulberry32）摇出等级/宜/忌/幸运物。小组件想显示同一张签，**把算法逐字搬到服务端**即可——同种子同算法，天然同源同签，不需要任何状态同步。

```js
// routes/widget.js —— 与前端 computeCard 完全一致
const rng = mulberry32(hashStr(`${date}|${who}|yanji-fortune-v1`));
const level = pickWeighted(rng, LEVELS);
const yi = pickN(rng, YI, 3);
```

两个要点：

1. **搬不动的部分就不搬**。签面的「寄语」有轮换池去重，seen 状态存在手机 localStorage，服务端拿不到。与其造一套同步，不如把它定性成产品设计：小组件只做预告，寄语留在 app 里当「拆信封」的部分——想看完整的签，就得打开 app。
2. **接口直接返回成品字符串**。`line1`/`line2` 是拼好的两行字，Tasker 拿到就显示，手机端零解析逻辑。显示端越笨，越不容易坏。

```json
{ "line1": "在一起第 275 天 · 今日小吉",
  "line2": "宜：把想说的说出口、开新脑洞 · 幸运物：亮着的小灯" }
```

⚠️ 代价是一条纪律：**以后改前端签池，服务端要同步改**，否则两边的签会悄悄分家。在两个文件头部互相写注释指路。

## Tasker 侧配置

一个任务两个动作，**独立建任务，别并进已有的上报任务**（触发时机不同、故障要隔离、以后改动不互相牵连）：

1. **HTTP Request**：GET 接口地址，Headers 填 `Authorization:Bearer <token>`，勾上 Structure Output（Tasker 自动拆 JSON，之后 `%http_data.line1` 直接可用）
2. **Widget v2 Setup**：小工具名称随意（桌面认领时用）、布局 **Table**、文本栏填两行变量、**Number of Columns 填 1**（单列 = 每行文本各占一行）

再配 Profile 定时触发（零点后换签 + 早上兜底跑一次），桌面长按 → 小部件 → Tasker Widget v2 拖出来认领，完事。

## 四个坑，按踩坑顺序

**坑一：401，但 token 明明是对的。**
先在服务器上用同一个 token 把接口自测三遍（本机、走域名、冒号后无空格），全通——嫌疑瞬间锁死在手机端。真凶：Tasker 的 Headers 栏一行一个头，**token 中间混进一个真回车就会在断行处截断**（输入框的软换行无害，真回车致命）。修法最省事的是从已有任务里复制现成的 Authorization 行。

**通用教训：链路两端各自可以自测时，先测服务端，把嫌疑压缩到一端再排查。**

**坑二：`Invalid layout type: 在一起第 275 天……`。**
报错把签文原样吐了回来——显示内容被填进了「布局」栏。布局栏要的是布局**类型**（这版 Tasker 没有纯 Text 布局，可选 Buttons/媒体/圆形图片/Table/Custom，两行文字用 Table 最省事），内容住在选完布局才出现的「文本」栏里。

**坑三：`Number of Columns: 请输入一些文本`。**
Table 布局的必填项。另一个小陷阱：点这一栏右侧的放大镜会弹出**变量选择器**，看起来像「没有手动填写的地方」——其实直接点标题下方的输入线敲 `1` 就行。

**坑四：任务跑了，不报错、不变绿、没反应。**
动作 2 前面挂着小三角——它在等「Ask To Add If Not Present」发出的系统级「添加小部件」弹窗被确认，而国产 ROM 大概率把这类弹窗拦掉了，于是永远等不到。别等：手动去桌面长按 → 拖 Widget v2 → 认领，再重跑任务立刻变绿。

## 美化：安卓玻璃感的真相

想要 iOS 那种毛玻璃？先泼冷水：**安卓小组件（RemoteViews）做不了磨砂模糊**。iOS 玻璃感 = 半透明 + 模糊，缺了模糊这一步，半透明白叠在饱和度高的照片壁纸上就是一层「奶」——发浊，不是通透。

能做的：

- **背景色**支持 `#AARRGGBB` 带透明度。alpha 阶梯法试：`#80FFFFFF`（50% 白，偏奶）→ `#66FFFFFF`（40%，我们的定稿）→ `#4DFFFFFF`（30%，更透但字的衬底变薄）。改一个数跑一遍，试错成本为零
- ⚠️ **Material You Colors 勾选会盖过自定义背景色**，要自定义必须先取消勾选；反过来，留着勾就是「随壁纸自动取色」，也是一种躺平式好看
- **字体**跟系统字体走，小组件嵌不了自定义字体——换系统字体它就跟着换
